## 🗺️ Compliance OS — Alfred's Building Priorities (Advisement)

Based on what you've described, here's how I'd think about this:

### The Core Differentiator to Protect
**Operational documented process first** — this is not how any current GRC tool is built. They all start with controls and map people to them. You start with how the org actually operates, then map *that* to frameworks. This is the right way. Don't let feature creep erode it.

### Framework Coverage to Prioritize (in order)
1. **NIST CSF 2.0** — universal, required for most US orgs
2. **SOC 2 Type II** — what startups/SaaS need first
3. **ISO 27001:2022** — enterprise and international market
4. **GDPR / CCPA** — privacy layer across all of the above
5. **HIPAA** — healthcare vertical unlock

### AI Engineering Angles for Compliance OS
- **Controls mapping AI** — auto-suggest framework mappings from uploaded policy documents
- **Gap analysis engine** — compare current state vs. target framework
- **Vendor risk scoring** — AI-assisted vendor questionnaire analysis
- **Evidence collection assistant** — Claude Code helping gather audit evidence
